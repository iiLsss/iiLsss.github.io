工厂模式

工厂模式是一种创建型设计模式，它定义了一个用于创建对象的接口，但由子类决定要实例化的类是哪一个。工厂模式使得一个类的实例化延迟到其子类。

示例： 在一个图形应用程序中，不同的形状（如圆形、方形）可以由不同的工厂方法创建，而不需要知道具体的形状类。

// 基础类 Shape
class Shape{}
// 圆形类
class Circle extends Shape{
  draw() {
    console.log('draw circle')
  }
}
// 方形类
class Square extends Shape{
  draw() {
    console.log('draw square')
  }
}
// 形状工厂类
class ShapeFactory{
  static getShape(type) {
    if (type === 'circle'){
      return new Circle()
    }
    if (type === 'square') {
      return new Square()
    }
    return null
  }
}
// 使用工厂模式
const circle = ShapeFactory.getShape('circle')
circle.draw() // draw circle
const square = ShapeFactory.getShape('square')
square.draw() // draw square

详细解释

  1. Shape 类：这是一个抽象类，定义了 draw 方法，该方法在子类中需要被重写。
  2. Circle 和 Square 类：这是具体的形状类，它们继承了 Shape 类并实现了 draw 方法，分别输出 "Drawing Circle" 和 "Drawing Square"。
  3. ShapeFactory 类：这是工厂类，它包含一个静态方法 getShape，根据传入的 shapeType 返回相应的形状对象实例。如果 shapeType 是 "CIRCLE"，则返回一个 Circle 对象；如果是 "SQUARE"，则返回一个 Square 对象；否则返回 null。
  4. 使用工厂模式：通过 ShapeFactory.getShape 方法，根据不同的 shapeType 获取相应的形状对象实例，并调用其 draw 方法。

示例：React.createElement 就是一个工厂函数，它通过调用 new Vnode 创建并返回一个 Vnode 对象

class Vnode {
  constructor(tag, attrs, children) {
    this.tag = tag;
    this.attrs = attrs;
    this.children = children;
  }
}

// 修正: 将 attr 改为 attrs
React.createElement = function(tag, attrs, children) {
  return new Vnode(tag, attrs, children);
}

// 示例使用
const vnode = React.createElement('div', { id: 'main' }, ['Hello, world!']);
console.log(vnode);
// 输出: Vnode { tag: 'div', attrs: { id: 'main' }, children: ['Hello, world!'] }
